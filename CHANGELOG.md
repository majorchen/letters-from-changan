## v0.1 — MVP（2026-06-04）

### 核心功能
- [x] 4身份选择（商人/乐师/游侠/书生），各有独立开场叙述和季节设定
- [x] 5.7秒电影式序幕（黑屏→标题→氛围语→背景图→打字机开场）
- [x] AI 流式叙事对话（Agnes agnes-2.0-flash）
- [x] 快捷选项按钮（根据剧情阶段动态变化）
- [x] 玩家状态持久化（localStorage：位置/章节/NPC关系/事件/信件历史）
- [x] 继续旅程 / 新游戏

### 信件系统
- [x] AI [MAILBOX] 标记触发邮箱发现（纯剧情驱动，不硬编码轮数）
- [x] 邮箱按钮出现时隐藏普通选项（不竞争）
- [x] 信纸 modal（旧纸质感，红印章，手写字体风格）
- [x] 2077年林深通信（独立 /api/letter，基于完整历史生成）
- [x] 回信功能（textarea，寄出后 AI 自动续叙述）
- [x] 关闭信件后自动续叙述
- [x] 信匣（📜图标，按时间倒序回看所有信件，信纸质感）
- [x] 后续来信：早期版本曾按回合触发；v0.5 改为玩家回信后后台准备，信件与视频完成后再通知

### 场景图系统
- [x] 4张预生成角色场景 webp（~250KB/张，无需等待API）
- [x] AI [SCENE:描述] 标记触发动态换图（任意地点，不受枚举限制）
- [x] 关键词兜底匹配（8个高频地点预设prompt）
- [x] 标记从显示内容中清除（玩家看不到）

### 视觉设计
- [x] 画卷式布局：场景图 fixed 顶部，文字浮于图片之上
- [x] 文字区 CSS mask 淡出（15%→30%），滚到图片区域时自然消融
- [x] 半透明紧凑顶部bar（backdrop-blur，显示位置/身份/次数/信匣）
- [x] 无气泡纯文字排版（AI叙述左对齐 amber-100，玩家话右对齐 amber-500）
- [x] 敦煌暖金色调（amber/stone 色系）
- [x] 首页全屏背景图（Agnes生成，webp 246KB）+ slow zoom + 浮动微粒 + staggered fade-in

### 基础设施
- [x] PWA manifest（standalone，深色主题）
- [x] 唐三彩邮箱 app icon（favicon 16/32 + apple-touch 180 + PWA 192/512）
- [x] 字体：系统宋体栈（Songti SC / SimSun / STSong / Noto Serif CJK SC），避免构建时依赖外部字体下载
- [x] 首版曾包含每日行动限制，v0.2 已移除
- [x] Agnes API key 通过 Vercel env vars 管理

#### 决定1：有限长度 > 无限沙盒
每条角色线约2小时游玩时长（约200-300轮对话，每轮AI回复简短100-200字）。不追求无限玩下去——好游戏是好故事好互动，不是无底洞。有限长度同时解决了AI记忆和结构两个问题：上下文窗口能覆盖大部分对话，三幕剧节奏有明确的起承转合。

#### 决定2：四线交叉（八方旅人/风花雪月模式）
4个角色（商人/乐师/游侠/书生）各自独立故事线，但共享同一个长安世界：
- **NPC交叉**：乐师线的主要NPC阿依，在游侠线里是酒肆偶遇的波斯女孩，她提到"认识一个弹琵琶的人"。玩家玩第二条线时意识到：那是上一条线的"我"。
- **世界记忆**：商人线帮王掌柜解决货源，书生线里王掌柜会提到"前阵子有个商人帮了大忙"。
- **信件交叉**：林深同时和四个时空的人通信。商人线里林深说"我还在给另一个人写信，他好像是个读书人"。四条线的林深是同一个林深。
- **碎片拼图**：每条线的林深透露不同的2077年信息（商人线→经济崩塌，书生线→文化断层，游侠线→战争，乐师线→AI替代创作）。四条线拼完才看到2077全貌。

单线体验完整，跨线有惊喜，全通有真相。

#### 决定3：AI记忆三层方案
现有 playerState 是"骨架记忆"（位置/NPC/事件），缺"叙事记忆"（NPC说了什么、玩家做了什么决定）。

- **第一层：叙事摘要（narrativeSummary）**——每N轮对话让AI总结"到目前为止发生了什么"（100字），存入 playerState，下轮注入 system prompt。成本极低，解决80%幻觉。
- **第二层：NPC独立记忆**——每个NPC一个小档案 { lastInteraction, attitude, knownFacts[] }，NPC不再每次见面像第一次认识你。
- **第三层：事件版本追踪**——不只存"发生了什么"，还存"谁说了什么版本"。例：{ event: "客栈对面的枣树", sources: { 林深: "很多年后还在", 王掌柜: "去年就砍了" } }。悬疑系统的基础设施。

#### 决定5：涌现式悬疑——AI的随机性是特色不是缺陷
AI叙事和手工编剧比精确度永远输。但AI能做一件手工编剧做不到的事：每个玩家的"真相"不同，因为世界为每个玩家生成了不同的矛盾。

核心机制：**不修复矛盾，标记矛盾**。当AI的叙述和之前不一致时（通过narrativeSummary检测），不纠正它，而是：
1. 矛盾日志里记一笔
2. 让李无名下次出场时说"这条街，昨天好像不是这样的"
3. 选项里出现"追问"动作（"你确定对面是面摊？我记得是棵枣树"），AI被迫回应矛盾，回应本身成为新碎片
4. 玩家参与了悬疑的构建，不只是发现——真相是玩家和AI共同生成的

前提：15-20个预埋锚点碎片不能丢（人工编写，不是AI生成）。锚点是不动的柱子，其余自由漂移。玩家在柱子之间看到的漂移，才会觉得"这里有什么不对"。全漂移=混乱，有骨架的漂移=悬疑。

这是只有AI互动叙事才能存在的新品类：Her Story的真相在玩家来之前就存在，来信长安的真相在玩家参与后才涌现。

#### 决定6：林深必须重写——从"人设卡"变成"人"
当前问题：林深"温暖、孤独、好奇"是一个设定，不是一个人。玩家更容易记住长安NPC（王掌柜讨价还价、阿依蹦波斯语、李无名说怪话），因为他们有行为、怪癖、摩擦。林深每封信都在"温暖地好奇"，太平了。

但林深不能去掉——没有信件，这就是AI唐朝行走模拟器，和AI Dungeon/Character.AI没有品类差异。信件是游戏的支点。

重写方向：
1. **具体的痛苦**：不是"孤独"这种抽象感受，而是一件具体的事（失去一个人/发现身世真相/被追踪）。他写信不是因为好奇，是因为这是他唯一的出口。
2. **会发脾气**：第四封信可以突然语气变了——"你上封信说的话让我很不舒服。你不懂。"玩家会愣住，然后认真想自己写了什么。这一刻林深变成了人。
3. **有个人秘密**：不是"2077年是什么样"的信息性秘密，是他自己的秘密。他为什么找到邮箱？他是不是刻意在找？他是不是知道的比说的多得多？
4. **会消失**：偶尔邮箱沉默很多轮。玩家开始想：他怎么了？是不是我说错了什么？沉默比一百封温暖的信更能建立连接。

核心原则：你记住的人，都是让你不舒服过的人。

#### 决定7：安史之乱倒计时 + 时辰系统（Nolan×小津，2026-06-05 确认）
天宝元年（742）距安史之乱（755）还有13年，但前兆已经出现：边境军报、物价波动、某些人开始囤货离京。林深从2077年回看，信里不经意提到"长安不会永远是这样的"。

两层效果同时存在，不矛盾：
- **Nolan层（倒计时）**：给叙事一个重力方向——玩家不是在闲逛，是在一个即将变化的世界里。第三幕应该让玩家感觉"时间不多了"。每个日常场景因为"这一切会消失"而更有重量。
- **小津层（时辰）**：prompt加入时辰流逝（清晨→午后→黄昏→夜晚），AI描写随时辰变化。西市清晨是卸货声和热馒头蒸气，傍晚是收摊吆喝和灯笼渐次亮起。NPC有"和你无关的生活在继续"的背景动作（王掌柜在里面骂伙计，你只是路过）。要有一个时刻玩家只是坐下来吃东西，不推任何剧情。

日常和危机同时存在：因为知道一切会消失，所以这顿饭更珍贵。

#### 决定8：关键时刻强制自由打字（Norman，2026-06-05 确认）
每条线设置2-3个"不给选项"的时刻——逼玩家自己打字回应。次数不多，但出现时有明确的文字指引（如"林深在信里问了你一个问题，你怎么回？"或NPC直视你等你说话）。目的：让玩家从"选择题模式"切换到"我是这个故事的参与者"，写出来的话会被AI记住并影响后续。

#### 决定9：视频作为叙事事件（Barlow，2026-06-05 确认）
视频不是"更好看的背景图"，是"夺走玩家控制权的时刻"。整条线最多3-4段，稀有才有力量。不可跳过，不可暂停——这几秒钟游戏拿走了你的控制权，这本身就是叙事。

**三种触发场景 × 三种嵌入方式：**

1. **回忆闪回**（林深信件触发，5-8秒）
   - 嵌入：信纸modal不关闭，背景纸纹crossfade到视频画面，结束后褪回纸纹。视频在文字下层，文字半透明仍可见。
   - 感受："信纸活了"，玩家怀疑自己是不是看错了。

2. **时空裂缝**（矛盾积累触发，2-3秒）
   - 嵌入：当前场景图原地glitch——长安街景和2077画面交替闪烁（hue-rotate+clip-path），恢复后叙述出现"你眨了眨眼。刚才好像有什么不对。"
   - 感受：不确定自己看到了什么。下次再发生时会紧张。不新增任何UI元素。

3. **结局时刻**（三幕收束，20-30秒）
   - 嵌入：文字区渐隐，输入框/选项/header消失，场景图crossfade到全屏视频。结束后fade to black，一行结语淡入（林深最后的话或"天宝十四载，安禄山起兵"），再淡入"返回首页"。
   - 感受：游戏不是"结束了"，是"安静下来了"。

**共同规则：**
- 没有播放按钮、进度条、全屏切换。视频在当前界面里"长出来"。
- 不打断布局：回忆在信纸里，裂缝在场景图里，结局是场景图扩展。
- 首次触发可加设备微震（navigator.vibrate，移动端），之后不震——让身体记住。

**技术：** Agnes video API 生成，单段约5秒。超过5秒的结局不做 ffmpeg，改为多段连续播放；完成后落 Supabase Storage，未 ready 时用 CSS 视觉事件兜底。

#### 决定4：节拍点代替预设结局
不写完整剧本，写"骨架节拍点"，AI在节拍之间自由生成（爵士乐模式：和弦进行固定，即兴自由）。

三层谜题结构：
1. **存在之谜**（已有）：邮箱是什么？林深是谁？信怎么穿越时空？
2. **信任之谜**（Phase 1 在做）：林深说的是真的吗？王掌柜和林深矛盾？李无名为什么知道邮箱？
3. **真相之谜**（Phase 2-3）：信里线索在长安有实物吗？四条线碎片拼出什么？

每条线内置：每3-5封信一个"认知翻转"（玩家以为的真相被推翻），每个NPC至少一个"矛盾时刻"，3-5个"里程碑发现"（碎片暗示而非直说，玩家自己拼）。
