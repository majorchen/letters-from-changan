# 来信长安：发布前收尾计划

> 确认日期：2026-06-06  
> 目标：完成可公开分享的完整体验，并以本文件作为本轮唯一实施基准。

## 一、最终体验原则

1. 登录不强制，游客可以直接游玩，本地保存；登录只负责跨设备同步。
2. 普通剧情、书信、场景图和视频各自有清晰职责，不互相泄漏内容。
3. 林深的每一封来信都必须附带一段约 5 秒的视频。
4. 视频在后台生成，通常约 2 分钟，最长按 5 分钟正常等待；不阻塞玩家继续探索长安。
5. 信匣保留现有全屏沉浸式界面，不改成底部面板。
6. 人物一致性只使用稳定的文字外貌档案，不传参考图。
7. 结局不生成视频，改为多幕文字落幕和电影式片尾。

## 二、任务清单

### Task 1：剧情选项与响应稳定性

- [x] 将 AI 选项改为更稳定、可解析的结构化输出。
- [x] 兼容现有旧格式，避免升级后破坏历史响应。
- [x] 选项必须引用当前人物、地点、物件或冲突，减少通用措辞。
- [x] 过滤最近两轮重复或高度相似的选项。
- [x] 解析失败时根据当前剧情生成具体兜底，不再高频出现“观察细节”“换个角度试探”。
- [x] 记录请求首字耗时和完整耗时。
- [x] 增加聊天超时和一次受控重试。

### Task 2：信件数据与通知状态

- [x] 每封信拥有独立 ID、发送者、正文、创建时间、已读状态和视频附件状态。
- [x] 普通剧情不得展示或概括信件正文。
- [x] 新信到达时，本轮选项只出现一次“信匣里有一封新信”。
- [x] 玩家若选择其他选项，该提示后续不再重复。
- [x] 未读信继续保留，右上角信匣图标保持柔光。
- [x] 新的一封信到达时，才再次出现一次来信选项。
- [x] 多封未读信互不覆盖，打开一封不能清除其他未读信。

### Task 3：Agnes 视频任务

- [x] 创建任务使用 `POST /v1/videos`。
- [x] 保存创建响应中的 `video_id` 与 `task_id`。
- [x] 优先使用 `GET /agnesapi?video_id=...&model_name=agnes-video-v2.0` 查询。
- [x] 仅对历史任务回退到 `GET /v1/videos/{task_id}`。
- [x] 支持从 `remixed_from_video_id`、`video_url`、`url` 和 `output` 提取视频地址。
- [x] 使用 121 帧、24fps，生成约 5 秒视频。
- [x] 轮询覆盖最长 5 分钟，并允许页面关闭后恢复查询。
- [x] 完成后写入 Supabase `story-videos` 和 `video_assets`。
- [x] 失败任务允许受控重试，不能永久卡住书信。

### Task 4：每封林深来信的视频附件

- [x] 第一封信使用预生成视频配置；若资源暂缺，使用同一任务机制生成并缓存。
- [x] 玩家寄出回信后立即启动下一封来信及其视频准备。
- [x] 信件与视频都完成后，才向玩家发出新信通知。
- [x] 每封林深来信必须有视频，玩家回信不附视频。
- [x] 视频生成期间不阻塞长安剧情。
- [x] 信笺内显示 16:9 视频首帧和明显播放按钮。
- [x] 点击后在原位置 `playsInline` 播放，不覆盖游戏、不跳转。

### Task 5：信匣与发光

- [x] 保留当前全屏沉浸式信匣。
- [x] 已读来信和玩家回信继续完整展示。
- [x] 未读来信显示“未启封”状态，点击后原位置展开。
- [x] 信匣顶部或列表中清晰标记未读数量。
- [x] 信匣图标发光改为圆形柔光，消除方形 `box-shadow` 边界。
- [x] 信匣图标存在未读时显示轻量金色圆点。

### Task 6：人物视觉一致性

- [x] 为四个玩家身份和核心 NPC 建立固定文字外貌档案。
- [x] 新命名 NPC 首次出现时生成并保存文字外貌档案。
- [x] 后续场景图自动复用对应人物档案。
- [x] 场景、动作、表情和光线可以变化，身份特征必须稳定。
- [x] 增加手脚、肢体、重复人物、重复脸、现代服装、文字和水印等负向约束。
- [x] 继续使用 Agnes 文生图，不传人物参考图。

### Task 7：多幕结局与电影片尾

- [x] 移除随机记忆视频、时空裂缝视频和结局视频。
- [x] 第三幕收束时生成结构化结局：结局名称、4 至 6 段落幕文字。
- [x] 复用首页背景图，以开场式淡入、停留、淡出连续呈现。
- [x] 结局内容回应角色身份、关键选择、林深关系、NPC 与最终判断。
- [x] 故事落幕后播放电影式片尾。
- [x] 片尾署名仅使用 `MajorChen | AI Fisher`。
- [x] 不显示邮箱，不显示 Agnes 官网。
- [x] 片尾展示 App Logo、产品名称、技术能力和简短创作初心。
- [x] 最底部只保留“重新开始”和“分享这段旅程”。
- [x] “重新开始”返回首页。
- [x] “分享这段旅程”复用信鸽分享流程，分享摘录固定使用故事落幕第一段。
